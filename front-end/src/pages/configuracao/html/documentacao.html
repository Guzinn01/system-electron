Apresentação de Projeto – Sistema de Gestão "Capitão Consórcios"

Versão: 1.1.0 (Projeto de Portfólio)

Data de Conclusão: Agosto de 2025

1. Introdução ao Projeto

1.1. Objetivo

Este documento apresenta o Sistema "Capitão Consórcios", um projeto de software completo desenvolvido para demonstrar competências em engenharia de software, incluindo desenvolvimento full-stack, arquitetura de sistemas, integração com APIs e análise de dados.

1.2. Visão Geral do Sistema

O "Capitão Consórcios" é uma aplicação de gestão integrada desenvolvida como um projeto de portfólio. Embora tenha sido idealizado para um cenário empresarial real, seu propósito principal é servir como uma vitrine de habilidades técnicas. A ferramenta centraliza o controle de clientes (CRM), registro de ponto, simulação de parcelas e análise de dados de sorteios. O acesso é gerenciado por um sistema de permissões, garantindo uma arquitetura segura e modular.

2. Funcionalidades e Demonstração de Habilidades

O sistema foi estruturado nos seguintes módulos, cada um projetado para resolver um problema de negócio e demonstrar uma capacidade técnica específica:

2.1. Simulador de Parcelas com Geração de PDF

Ferramenta para simular valores de parcelas de consórcio. A funcionalidade de exportar a simulação para um arquivo PDF demonstra a capacidade de gerar relatórios dinâmicos e formatados no lado do servidor ou cliente.

2.2. Gestão de Clientes (CRUD Completo)

(Acesso restrito por permissão). Implementação de um módulo CRUD (Create, Read, Update, Delete) para gerenciar a carteira de clientes, demonstrando o domínio sobre operações de banco de dados e manutenção de registros.

2.3. Análise de Lances com Integração de API

(Acesso restrito por permissão). O módulo mais complexo do sistema, focado em Business Intelligence (BI) e análise de dados:

  • Visualização de Dados: Exibe sumários, médias e históricos de lances em gráficos e tabelas, utilizando paginação e filtros para uma performance otimizada na consulta de grandes volumes de dados.
  • Upload e Processamento de Dados: Permite o upload de planilhas (.xlsx) que são processadas por uma API dedicada (desenvolvida em Python). Essa API é responsável por validar, processar e salvar milhares de registros de forma segura e eficiente no banco de dados, evitando duplicatas e garantindo a integridade dos dados.

2.4. Módulos de Apoio

Funcionalidades adicionais como Registro de Ponto e Consulta de Sorteios foram implementadas para criar um ecossistema de gestão mais completo e robusto.

2.5. Sistema de Autenticação e Permissões

O sistema conta com uma tela de login segura e um controle de acesso baseado em permissões, onde administradores podem gerenciar outros usuários, demonstrando a implementação de lógicas de segurança e diferentes níveis de acesso.

3. Documentação Técnica da API

Para demonstrar a organização do back-end, a API de Análise de Lances possui uma documentação técnica interativa (gerada via Swagger/OpenAPI, por exemplo). Ela permite a visualização de endpoints, modelos de dados e a realização de testes diretamente pela interface.
